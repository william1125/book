LVM 扩展

发表于 2019-03-11 | 分类于 linux系统

1、查看新增磁盘编号

fdisk -l

2、对新增磁盘分区和格式化

fdisk /dev/xvdc
#按n创建分区
#按p创建主分区
#其他默认
#完成后按w保存
mkfs.ext4 /dev/xvdc1

3、创建PV

pvcreate /dev/xvdc

4、把磁盘添加到VG组

vgdisplay
#查看当前VG名称
vgextend centos /dev/xvdc1

4、扩展LV

lvdisplay
#查看当前LV
lvextend -l +100%FREE /dev/centos/root

5、重新加载逻辑卷的大小

resize2fs /dev/centos/root
#重新加载LVM分区大小
#如果报错，可尝试执行
xfs_growfs /dev/centos/root

自定义nginx证书

发表于 2019-03-08 | 更新于 2019-03-12

自定义证书配置

一、生成证书

1、进入生成证书目录

cd /etc/nginx/

2、创建服务器私钥，命令会让你输入一个口令：（这里输入一个指定密码，输入两次）

openssl genrsa -des3 -out server.key 1024

3、创建签名请求的证书（CSR）：

openssl req -new -key server.key -out server.csr

A位置输入CN，B位置输入服务器IP地址（也可以不输入），其他位置可以为空

%1、在加载SSL支持的Nginx并使用上述私钥时除去必须的口令：

cp server.key server.key.org
openssl rsa -in server.key.org -out server.key

5、最后标记证书使用上述私钥和CSR：

openssl x509 -req -days 365 -in server.csr -signkey server.key -out server.crt

二、修改nginx配置文件

在nginx配置文件中加入下面内容，/etc/nginx为证书实际路径

server {

listen 443;

server_name localhost;

ssl on; ssl_certificate /etc/nginx/server.crt;

ssl_certificate_key /etc/nginx/server.key;

ssl_session_timeout 5m;

ssl_protocols SSLv2 SSLv3 TLSv1;

ssl_ciphers ALL:!ADH:!EXPORT56:RC4+RSA:+HIGH:+MEDIUM:+LOW:+SSLv2:+EXP;

ssl_prefer_server_ciphers on;

}

三、测试

1、重新加载配置
systemctl reload nginx

2、访问地址

https://IP
